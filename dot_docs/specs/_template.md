# 仕様名

## 配置

このテンプレートから作る仕様は、対象リポジトリのルート（`jj root`）から見た `docs/specs/` に置く。

## 目的

この変更で達成したいユーザー価値、運用上の狙い、または問題設定を書く。

## 受け入れ基準

### AC-1：最初の完了条件

- 入力 → 結果：具体的な入力や操作を書くと、期待される観測可能な結果を書く

### AC-2 (Depends On：AC-1)：依存のある完了条件

- 入力 → 結果：AC-1 が完了した状態で具体的な入力や操作を書くと、期待される観測可能な結果を書く

## ネガティブ要件

- この仕様で実装しないことを書く
- 既存挙動として残すものを書く
- 外部連携、永続化、抽象化など、過剰実装になりやすい禁止事項を書く

## スコープ上限

想定変更量は N ファイル前後、総行数の目安は N 行以内。これを大きく超えたら軸Cで超過候補として扱う。

## 技術的制約

- `docs/specs` 以下には仕様に関連する技術制約を書く
- 全タスク共通の規約は `CLAUDE.md` に書く

## タスク一覧

- [ ] AC-1：最初の完了条件
- [ ] AC-2：依存のある完了条件

進行中のタスクは `- [ ] AC-N：（進行中）タスク名`、完了したタスクは `- [x] AC-N：タスク名` と書く。

## 検証ゲート

- 軸A：受け入れ基準に対応するテストまたは存在チェックが成功している
- 軸B：diff がネガティブ要件に反していない
- 軸C：diff 量がスコープ上限を大きく超えていない
- 軸D：軸Bまたは軸Cの候補がある場合だけ、人間が最終判定する
