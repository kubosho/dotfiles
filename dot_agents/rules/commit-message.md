# Commit Messages

Applies when authoring commit messages in any VCS.

## Outcome

Each commit conveys the motivation for the change in the language convention of recent project history, so the history alone reveals intent.

## Constraints

- Match the message language (subject and body) to the last 5 commit logs in the repository. The same language convention also applies to code comments authored as part of the same change.
- The body explains "why" (the reason or motivation), not just "what". The diff already shows what.
- Do not introduce a new language convention without explicit user instruction.
